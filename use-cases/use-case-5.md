# USE CASE: 5 Add a New Employee's Details

## CHARACTERISTIC INFORMATION

### Goal in Context

As an *HR advisor* I want to *add a new employee's details* so that *I can ensure the new employee is paid*

### Scope

Employee Management System.

### Level

Primary task.

### Preconditions

We know all the employee's details. We have admin access to the database.

### Success End Condition

Employee details are successfully added to the database.

### Failed End Condition

No employee details are added.

### Primary Actor

HR Advisor.

### Trigger

A new employee has been hired and must be added to the Employee Management System.

## MAIN SUCCESS SCENARIO

1. HR advisor logs into the Employee Management System.
2. HR advisor selects option to add employee into the system.
3. HR advisor enters employee information.
4. HR advisor reviews info and adds employee.
5. HR advisor confirms employee addition and exits system.

## EXTENSIONS

4. **Entered Information is Incorrect**:
   1. HR advisor revises entered information.
   2. HR advisor adds employee.

## SUB-VARIATIONS

None.
