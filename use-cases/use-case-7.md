# USE CASE: 7 Update Employee's Details

## CHARACTERISTIC INFORMATION

### Goal in Context

As an *HR advisor* I want to *update an employee's details* so that *employee's details are kept up-to-date.*

### Scope

Employee Management System.

### Level

Primary task.

### Preconditions

We know the employee's name. Employee exists in the database.

### Success End Condition

Employee details are successfully updated.

### Failed End Condition

Employee details in database remain the same.

### Primary Actor

HR Advisor.

### Trigger

An employee's information has changed.

## MAIN SUCCESS SCENARIO

1. HR advisor selects option to edit employee information.
2. HR advisor enters updated employee information.
3. HR advisor reviews info.
4. HR advisor confirms update.

## EXTENSIONS

3. **Entered Information is Incorrect**:
   1. HR advisor revises entered information.
   2. HR advisor updates employee.

## SUB-VARIATIONS

None.
